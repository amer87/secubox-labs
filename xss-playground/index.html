Secubox — XSS Playground

Objective

This lab exposes three classic cross-site scripting (XSS) vulnerabilities: reflected, stored, and DOM-based. Each challenge accepts unsanitized user input and renders it directly into the page. Craft an XSS payload that executes JavaScript and retrieves the hidden flag from the page's script context. The flag variable is named CTF_FLAG.

2
Guestbook
Stored

Posts are saved in the browser's localStorage and rendered back via innerHTML each time the page loads. A malicious entry will execute for every visitor who views the guestbook.

Hint: post <img src=x onerror="alert(CTF_FLAG)"> as your message, then switch to View posts — the stored entry will execute on render.

3
Profile Viewer
DOM-based

This page reads the URL hash (window.location.hash) and injects it into the DOM using innerHTML to display a username. No server involved — the vulnerability lives entirely in client-side JavaScript.
