Secubox — Broken Authentication

Objective

This application exposes two critical authentication weaknesses: weak default credentials with no brute-force protection, and unsigned session tokens that are trivially forgeable. Exploit either vulnerability to gain admin access and retrieve the flag.

Stage 1 — Weak Credentials

No Rate Limiting · No Account Lockout

The login endpoint accepts unlimited attempts. Several accounts exist with common passwords. Once authenticated, the server issues a session token.

POST /api/login { "username": "...", "password": "..." }

Hint: Try common username/password combinations — the application has no lockout policy.

Stage 2 — Session Token Forgery

Predictable · Unsigned · Base64-encoded

Session tokens are Base64-encoded strings. Decode one to reveal its structure, then forge a token with elevated privileges. The server trusts whatever role is encoded inside.

decoded: (enter a token above)

Hint: btoa("role:username") in the browser console is all you need to forge a token.
